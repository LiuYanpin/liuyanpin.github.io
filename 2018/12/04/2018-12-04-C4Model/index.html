C4 Model / C4 模型

大道至简
2018-12-04

C4 代表上下文（Context）、容器（Container）、组件（Component）和代码（Code）。
C4 Model要解决的问题：
用来描述软件架构。在不同的细节层面。
几个层面：

  • Person
  • Software System
    最高级的抽象。描述给用户的交付价值。包括你在建模的软件系统和此软件系统依赖的系统。
  • Container
    为了全局的系统能工作而运行的一个东西。
    比如一个服务器端的应用程序，客户端的应用程序。移动App。
    数据库、文件系统等。
    Container是一个边界：上下文执行和数据存储的一个边界。
    一般来说部署和运行时环境是不一样的。
  • Component
    定义时一组有关的包装好的接口。
    Java中可以认为是一个接口的实现类。
