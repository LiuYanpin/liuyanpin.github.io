IAM

IAM - Identity and Access Management

MFA

Multi-factor authentication for highly privileged users
MFA：Multi-Factor Authentication (MFA) 是一种简单有效的最佳安全实践方法,它能够在用户名和密码之外再额外增加一层安全保护。

S3

S3 - Simple Storage Service
静态内容存储，Web App Reference Architecture。
大数据存储备份，Stream Data Architecture。

最佳实践

  • 贯彻最小权限原则（Principle of Least Privilege）
  • 隐藏您的 AWS 账户根用户访问密钥, 创建单独的 IAM 用户
  • 使用组向 IAM 用户分配权限
  • 尽量使用由 AWS 定义的策略分配权限
  • 为您的用户配置强密码策略，不共享访问密钥
  • 为特权用户启用 MFA
  • 针对在 Amazon EC2 实例上运行的应用程序使用角色来委托权限
  • 删除不需要的凭证，定期轮换凭证

Vagrant

Vagrant 是一个用来构建和管理虚拟机环境的工具。
Vagrant典型使用案例是以简单并且一致的方式构建工作或者开发环境。降低了开发者搭建环境的时间，提高了生产力。解决了“在我的机器上可以工作”的问题。
Vagrant 是为了方便的实现虚拟化环境而设计的，使用 Ruby 开发，基于 VirtualBox 等虚拟机管理软件的接口，提供了一个可配置、轻量级的便携式虚拟开发环境。

CloudFormation

CloudFormation给予了用户一种简单的方法来创建和管理一系列有关联的AWS的资源，可以有序的及可预见的初始化和更新这些资源。
Template只是一个yml或json格式的文件，如果想要使用它的话，需要创建一个Stack，在Stack中指定需要的Template，然后AWS才会按照Template中的定义来创建及初始化资源。
