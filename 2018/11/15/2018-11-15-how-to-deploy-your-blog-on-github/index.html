How to Deploy Your Blog on GitHub

Nov 15, 2018GitHubHits

I spend a whole day today thinking out how could I build my own blog rather than using other website.
Last night I found out a blog from a fellow student of my old school.
So I learned to build my own blog today.
Here are my steps below.

Step1 Before

Before you start, go to MyBlog and click star or fork .

Step2 Choose a framework

I choose hexo as my blog framework.

  • It support markdown.
  • It’s fast to build.
  • It’s eazy to deploy.

  • Step3 Init environment

  • make sure you have installed Node.js

  • make sure you have installed Git

Step4 Writing blog

Always remember that _config.yml is your most important config file. You can config many things here. And it’s written in YAML.
If you want to write a new blog, use hexo new first-blog and it will generate a new markdown file named first-blog.md under folder source/_posts

Step5 See your blog

Then you can see you first blog website on http://localhost:4000/

Step6 Deploy your website on GitHub Pages

  • new a repository on your github, name <your github username>.github.io, eg, liuyanpin.github.io
  • see setting, go to GitHub Pages, you will find it automatically enabled your github pages function.
  • go to your MyBlog folder on your computer, go to public folder, then run git init

  • add a remote repo for your public folder.

  • eg,

  • git push -u origin master push your local website to remote repo.

  • make sure your xxx.github.io.git have index.html file, since it’s the entrance of your website.

Step7 Change your theme

  • go to https://hexo.io/themes/
  • find a theme that you like, eg, BlueLake, click it to its repo on GitHub. hexo-theme-BlueLake
  • go to your root folder, eg, MyBlog
  • do as the readme told you.

Step8 Maintain your website

  • It’s important to maintain your own website.
