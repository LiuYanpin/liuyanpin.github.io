关于SQL

Nov 21, 2018Hits

本文是摘自W3C的有关SQL中需要注意的地方。

  • SQL是什么
    SQL 是用于访问和处理数据库的标准的计算机语言。
  • SQL 对大小写不敏感
  • SQL 使用单引号来环绕文本值（大部分数据库系统也接受双引号）。如果是数值，请不要使用引号。

  • 并非所有的数据库系统都支持 TOP 子句

  • MySQL

  • LIKE "%"可用于定义通配符（模式中缺少的字母）。

  • 不同的数据库对 BETWEEN...AND 操作符的处理方式是有差异的。
  • AS别名（表和列）
  • INNER JOIN 与 JOIN 是相同的
  • INNER JOIN 关键字在表中存在至少一个匹配时返回行。
  • LEFT JOIN 关键字会从左表那里返回所有的行，即使在右表中没有匹配的行。
  • RIGHT JOIN 关键字会右表那里返回所有的行，即使在左表中没有匹配的行。
  • 在某些数据库中， LEFT JOIN 称为 LEFT OUTER JOIN。RIGHT JOIN 称为 RIGHT OUTER JOIN。
  • 只要其中某个表存在匹配，FULL JOIN 关键字就会返回行。在某些数据库中， FULL JOIN 称为 FULL OUTER JOIN。
  • FULL JOIN 关键字会从左表 (Persons) 和右表 (Orders) 那里返回所有的行。如果 “Persons” 中的行在表 “Orders” 中没有匹配，或者如果 “Orders” 中的行在表 “Persons” 中没有匹配，这些行同样会列出。
  • SELECT INTO 语句从一个表中选取数据，然后把数据插入另一个表中。
  • 约束：NOT NULL 约束强制列不接受 NULL 值。
    UNIQUE 约束唯一标识数据库表中的每条记录。
    PRIMARY KEY 拥有自动定义的 UNIQUE 约束。
    PRIMARY KEY 约束唯一标识数据库表中的每条记录。
    FOREIGN KEY 指向另一个表中的 PRIMARY KEY。
    CHECK 约束用于限制列中的值的范围。
    DEFAULT 约束用于向列中插入默认值。
  • CREATE INDEX在表中创建索引，以便更加快速高效地查询数据。
  • Auto-increment 会在新记录插入表中时生成一个唯一的数字，不需要手动增加。
  • 无法使用比较运算符来测试 NULL 值，比如=, <, 或者 <>必须使用 IS NULL 和 IS NOT NULL
  • 函数基本类型：Aggregate functions
  • 在 SQL 中增加 HAVING 子句原因是，WHERE 关键字无法与合计函数一起使用。
